# Quality Assurance

Track ini membangun kemampuan memastikan kualitas produk melalui pengujian sistematis. Kamu akan belajar test case design, automation testing, API testing, dan security testing. Cocok untuk yang ingin jadi QA Engineer, Test Automation Engineer, atau QA Lead.

**Kompetensi per Level:**

- Beginner: 5 kompetensi
- Intermediate: 4 kompetensi
- Advanced: 3 kompetensi

## Level: Beginner

### Testing Types & Pyramid

- **Mastery Level:** Level 1 - Understand & Articulate
- **Evidence:** Penjelasan konsep perbedaan Unit, Integration, dan E2E Testing
- **Exit Criteria:** Menunjukkan pemahaman teoritis mengenai efisiensi piramida testing

### Manual Test Case Writing

- **Mastery Level:** Level 2 - Apply & Execute
- **Evidence:** Daftar test case manual (Excel/Notion) mencakup skenario positif dan negatif
- **Exit Criteria:** Skenario uji terstruktur dengan langkah-langkah ekspektasi yang jelas dan konsisten

### Bug Reporting & Triage

- **Mastery Level:** Level 2 - Apply & Execute
- **Evidence:** Laporan bug di platform pelacakan (Jira/Linear) yang memuat screenshot dan log error
- **Exit Criteria:** Developer dapat mereproduksi bug tersebut hanya dengan mengikuti laporan QA

### Exploratory Testing

- **Mastery Level:** Level 2 - Apply & Execute
- **Evidence:** Laporan sesi exploratory testing terhadap area berisiko tinggi
- **Exit Criteria:** Menemukan celah visual atau fungsional di luar test case standar

### Acceptance Testing (UAT)

- **Mastery Level:** Level 2 - Apply & Execute
- **Evidence:** Laporan hasil uji penerimaan pengguna (UAT) bersama PM
- **Exit Criteria:** Memverifikasi keselarasan fitur dengan Acceptance Criteria sebelum masuk produksi

## Level: Intermediate

### Test Automation Fundamentals

- **Mastery Level:** Level 2 - Apply & Execute
- **Evidence:** Script automation testing dasar (Cypress, Playwright, Selenium)
- **Exit Criteria:** Test berjalan otomatis untuk mensimulasikan alur login pengguna

### API Testing

- **Mastery Level:** Level 2 - Apply & Execute
- **Evidence:** Koleksi Postman atau script Newman untuk menguji status response API
- **Exit Criteria:** Memvalidasi data schema dan response code 200, 400, dan 500 secara otomatis

### Performance & Load Testing

- **Mastery Level:** Level 3 - Evaluate & Architect
- **Evidence:** Laporan load testing menggunakan K6 atau JMeter
- **Exit Criteria:** Menemukan batas maksimum kapasitas request per second (RPS) sebelum server crash

### Regression Testing Strategy

- **Mastery Level:** Level 3 - Evaluate & Architect
- **Evidence:** Rencana uji regresi yang efisien untuk setiap rilis berkala
- **Exit Criteria:** Mengurangi durasi manual test tanpa melewatkan celah regresi

## Level: Advanced

### QA Architecture & Strategy

- **Mastery Level:** Level 3 - Evaluate & Architect
- **Evidence:** Dokumen strategi QA organisasi (pemilihan tools, integrasi pipeline)
- **Exit Criteria:** Mengurangi defect leakage rate di produksi secara terukur

### Security Testing Fundamentals

- **Mastery Level:** Level 3 - Evaluate & Architect
- **Evidence:** Laporan vulnerability scanning (OWASP ZAP)
- **Exit Criteria:** Menemukan dan mendokumentasikan celah keamanan dasar (SQLi/XSS)

### Quality Metrics & Reporting

- **Mastery Level:** Level 4 - Peer Instruction & Knowledge Share
- **Evidence:** Laporan metrik kualitas berkala (Defect Density, Test Pass Rate)
- **Exit Criteria:** Menyajikan kualitas produk berbasis data objektif ke pihak manajemen
