# Brief Project - YYZU Batch 1

> Brief resmi untuk Mini Project dan Collaborative Project YYZU Batch 1.

---

## Mini Project

### Apa itu Mini Project?

Mini Project adalah project tim (2-4 orang, bisa 1 track saja) yang dikerjakan selama 6 minggu (Minggu 3-8). Setiap tim membuat project dari awal sesuai track masing-masing. Ide project adalah tanggung jawab tim.

**Tema WAJIB:** Economic DAN/ATAU Education.

Project harus relevan dengan salah satu atau kedua tema tersebut. Ide bebas, asalkan berada dalam koridor tema. Satu ide per tim.

### Durasi

- **Mulai:** Minggu ke-3 (29 Juli 2026, MP Kickoff)
- **Selesai:** Minggu ke-8 (6 September 2026)
- **Total:** 6 minggu pengerjaan

### Deliverable per Track (per tim)

| Track | Deliverable |
|-------|-------------|
| **Web Dev** | Working website (deployed) |
| **UI/UX** | Prototype di Figma + design spec |
| **PM** | PRD lengkap + user story map |
| **QA** | Test plan + test cases + test report |
| **Data & AI** | Dataset analysis + visualisasi + insight |
| **Software Dev** | Working application |
| **Mobile** | Working mobile app/prototype |
| **Creative Technology** | Interactive content |
| **DevOps & Cloud** | CI/CD pipeline atau infra setup |
| **Digital Product** | Product spec + business case |
| **Technology Collaboration** | Collaboration framework doc |

### Kriteria Penilaian (Binary Pass/Fail)

1. **Fungsi dasar bekerja.** Project bisa dijalankan dan fungsi utamanya beroperasi.
2. **Kode/desain rapi.** Struktur kode atau desain tertata, bukan sekadar jalan.
3. **Dokumentasi lengkap.** README, deskripsi project, cara menjalankan, dan catatan penting tersedia.
4. **Menggunakan tools standar YYZU.** GitHub, Notion, Figma (sesuai kebutuhan track).
5. **Relevan dengan tema Economic/Education.** Project jelas terhubung dengan salah satu atau kedua tema.
6. **Kerja tim terbukti.** Evidence berupa git log, task board, atau pembagian tugas jelas antar anggota tim.

### Deadline

**6 September 2026**

Tidak ada perpanjangan waktu. Pastikan project selesai dan ter-deploy/selesai sebelum batas waktu.

## Collaborative Project

### Apa itu Collaborative Project?

Collaborative Project adalah project tim kolaboratif yang dikerjakan selama 10 minggu (Minggu 9-18). Tim terdiri dari 4 orang (1 Project Leader + 3 anggota, role bebas disesuaikan kebutuhan project). Problem statement adalah tanggung jawab tim.

**Tema WAJIB:** Economic DAN/ATAU Education.

Tim dibentuk oleh Divisi PM. Komposisi tim **wajib lintas track** (minimal 2 track berbeda dalam 1 tim).

### Durasi

- **Mulai:** Minggu ke-9 (9 September 2026, CP Kickoff)
- **Selesai:** Minggu ke-16 (31 Oktober 2026, Demo Day)
- **Total:** 10 minggu pengerjaan

### Komposisi Tim

Tim terdiri dari 4 orang dengan role berikut:

| Role | Tanggung Jawab |
|------|----------------|
| **Project Leader** | Koordinasi tim, fasilitasi sprint ceremonies, pastikan timeline tercapai |
| **Anggota 1** | Kontribusi sesuai keahlian track masing-masing |
| **Anggota 2** | Kontribusi sesuai keahlian track masing-masing |
| **Anggota 3** | Kontribusi sesuai keahlian track masing-masing |

**Role anggota tim bebas, disesuaikan kebutuhan project. Contoh komposisi:**
- 2 Developer + 1 Designer
- 1 Developer + 1 Designer + 1 QA
- 3 Developer (jika project teknis)

Komposisi tim **wajib lintas track** — anggota tim WAJIB berasal dari track berbeda (minimal 2 track berbeda dalam 1 tim). Satu orang bisa merangkap role sesuai kebutuhan.

### Deliverable

1. **Working product (MVP).** Produk fungsional yang menyelesaikan problem statement tim.
2. **Presentasi Demo Day.** Presentasi produk di hadapan juri dan peserta lain.

### Kriteria Penilaian (Binary Pass/Fail)

1. **Product fungsional (MVP).** Produk berjalan dan menyelesasikan masalah yang didefinisikan.
2. **Kerja tim terbukti.** Evidence berupa git log, task board, dan standup log.
3. **Sprint ceremonies terlaksana.** Sprint planning, standup, review, dan retrospective dilakukan secara konsisten.
4. **Presentasi jelas dan profesional.** Penyampaian terstruktur, demo berjalan lancar, pertanyaan dijawab dengan baik.
5. **Dokumentasi lengkap.** README, arsitektur, cara menjalankan, dan catatan penting tersedia.
6. **Relevan dengan tema.** Produk jelas terhubung dengan tema Economic dan/atau Education.

### Deadline Presentasi

**Demo Day: 31 Oktober 2026**

Tim wajib mempresentasikan produk di Demo Day. Tidak ada presentasi ulang.

---

## Catatan Umum

- Semua project harus menggunakan repository GitHub sebagai version control.
- Progress dilaporkan melalui Kanban Board dan standup harian.
- Ide project adalah tanggung jawab tim. Divisi PM tidak menyediakan ide.
- Tema Economic dan Education bersifat WAJIB, bukan opsional.
- Kolaborasi adalah esensi dari Collaborative Project. Manfaatkan keahlian masing-masing anggota dalam tim.
