description: "Kalender event 18 minggu per batch yang mencakup detail event, aturan penjadwalan, dan definisi kegiatan."
---

Kalender event ini adalah panduan bagi Divisi Event & Planning dalam merencanakan kegiatan sepanjang siklus batch. Setiap batch berdurasi **4 bulan (18 minggu)** sesuai ketentuan AD/ART YYZU.
**Struktur batch:**
- **Minggu 1-2:** Learning Sprint (2 minggu) (Onboarding + belajar tools)
- **Minggu 3-8:** Mini Project (6 minggu)
- **Minggu 9-18:** Collaborative Project (10 minggu)
**Pola mingguan:** Setiap minggu memiliki **1-2 hari event** (tergantung fase project). Event dijalankan di hari yang berbeda dalam minggu yang sama.
**Mengapa jumlah hari event berbeda per fase?**
- **Fase Foundation (2 hari event):** Tim baru mulai, butuh banyak guidance dan workshop untuk membangun fondasi. Dua hari event memastikan ada cukup waktu untuk belajar tools dan konsep dasar.
- **Fase Discovery (1 hari event):** Tim sedang eksplorasi dan riset. Satu hari event cukup untuk check-in, sisa waktu digunakan untuk riset mandiri.
- **Fase Execution (1 hari event):** Tim fokus mengerjakan project. Terlalu banyak event justru mengganggu produktivitas. Satu hari event untuk sprint review dan diskusi teknis.
- **Fase Review (2 hari event):** Tim perlu presentasi, menerima feedback, dan melakukan revisi. Dua hari event memastikan ada waktu untuk review dan iterasi.
**Default waktu:**
- **Hari 1:** Selasa malam (20:00 WIB)
- **Hari 2:** Kamis malam (20:00 WIB)
- **Event panjang** (Workshop, Onboarding, Demo Day): Sabtu sore (16:00 WIB)
**Batas durasi:** Maksimal 150 menit (2.5 jam) per event.

## Aturan Penjadwalan
1. **Cek kalender ekosistem** sebelum menetapkan jadwal - hindari bentrok dengan event divisi lain.
2. **Minimal H-7** untuk event wajib, **H-14** untuk event opsional.
3. **Libatkan Koordinator Umum** dalam penetapan jadwal event yang melibatkan narasumber eksternal.
4. **Dokumentasikan perubahan jadwal** di Notion dengan alasan yang jelas.
5. **Maksimal 2 event per minggu** untuk menghindari kelelahan member.
6. **Event paralel harus beda hari** - Hari 1 dan Hari 2 tidak boleh di hari yang sama.
7. **Default waktu sore/malam** - hindari jadwal pagi/siang karena bentrok dengan kuliah/kerja member.

## Definisi Event (Glossary)
Definisi setiap jenis event yang ada di kalender ini.
### Onboarding Session
Sesi pengenalan untuk member baru. Berisi pengenalan ekosistem YYZU, struktur organisasi, cara kerja, tools yang digunakan, dan ekspektasi selama batch. Dipimpin oleh Divisi SDM dan Event.
### Learning Sprint Kickoff
Sesi pembukaan fase belajar. Berisi penjelasan kurikulum, timeline learning sprint, dan ekspektasi kompetensi yang harus dicapai sebelum project dimulai.
### Study Group
Sesi belajar bersama yang difasilitasi oleh anggota ekosistem. Fokus pada topik teknis spesifik sesuai track (Web Dev, UI/UX, PM). Format: presentasi + diskusi + latihan.
### Workshop
Sesi hands-on terstruktur dengan instruktur. Fokus pada penguasaan tool atau konsep tertentu. Format: presentasi + praktik langsung. Durasi lebih panjang dari Study Group.
### Industry Sharing
Sesi berbagi pengalaman dari praktisi industri. Berisi cerita karier, insight dunia kerja, atau tren teknologi terkini. Narasumber bisa dari Mentor, jaringan Founder, jaringan member, alumni, atau mitra industri.
### Technical Discussion
Diskusi teknis terfokus antara anggota ekosistem. Topik spesifik yang relevan dengan project atau teknologi yang sedang dipelajari. Format: presentasi singkat + diskusi terbuka.
### Workflow Simulation
Simulasi proses kerja industri dalam format terkontrol. Contoh: simulasi sprint planning, simulasi code review, simulasi pitching ke client. Tujuan: membiasakan member dengan workflow sebelum project dimulai.
### Mini Project Kickoff
Sesi pembukaan fase Mini Project. Berisi pengumuman tema, pembentukan kelompok, penjelasan timeline, dan ekspektasi deliverable.
### Collaborative Project Kickoff
Sesi pembukaan fase Collaborative Project. Berisi pengumuman tema, pembentukan kelompok lintas track, penjelasan timeline, dan ekspektasi deliverable.
### Code Review Session
Sesi review kode yang dilakukan di luar konteks project aktif. Bisa menggunakan kode pribadi member, kode lama, atau kode latihan. Tujuan: melatih kemampuan membaca dan mengevaluasi kode orang lain.
### Code/Design Review
Sesi review checkpoint untuk project yang sedang berjalan. Mentor atau senior member mengevaluasi kualitas kode/desain tim. Tujuan: memberikan feedback sebelum project terlalu jauh.
### Sprint Review + Retro
Sesi evaluasi akhir sprint. Tim mendemonstrasikan hasil kerja ke Mentor (Review), lalu melakukan evaluasi internal (Retrospective). Format: demo + feedback + refleksi.
### Product Discussion
Diskusi berfokus pada product thinking: validasi ide, analisis use case, product strategy. Berbeda dari Technical Discussion karena perspektifnya adalah produk, bukan implementasi teknis.
### Portfolio Review
Review menyeluruh terhadap portofolio member. Mencakup website portfolio, GitHub profile, atau kumpulan project. Tujuan: memberikan feedback untuk persiapan rekrutmen.
### Design Review Session
Sesi review khusus untuk desain UI/UX. Desainer mendapatkan feedback dari Mentor atau senior member tentang kualitas desain, konsistensi, dan usability.
### Career Preparation
Sesi persiapan karir untuk member. Berisi CV review, mock interview, portfolio critique, atau sesi career roadmap. Tujuan: mempersiapkan member memasuki dunia kerja.
### Mentor Sharing
Sesi berbagi pengalaman dari Mentor YYZU. Berisi cerita karier, insight dunia kerja, atau tren teknologi terkini. Di Batch 1, Mentor menjadi narasumber utama karena belum ada alumni. Ke depannya bisa diganti dengan Alumni Sharing setelah Batch 1 lulus.
### Rehearsal Demo Day
Sesi latihan presentasi sebelum Demo Day. Tim mempresentasikan proyek mereka dan mendapatkan feedback tentang cara penyampaian, struktur presentasi, dan handling pertanyaan.
### Demo Day
Sesi presentasi akhir proyek. Setiap tim mendemonstrasikan hasil kerja di hadapan Founder, Mentor, dan seluruh member YYZU. Ini adalah milestone akhir dari batch.
### End-of-Batch Feedback Session
Sesi refleksi dan feedback di akhir batch. Member memberikan input tentang kualitas program, saran perbaikan, dan pengalaman selama batch. Format: survei + diskusi terbuka.
### Batch Showcase + Closing
Sesi penutupan batch. Berisi showcase hasil kerja seluruh tim, penghargaan, dan penutupan resmi. Dipimpin oleh Event dan Koordinator Umum.
