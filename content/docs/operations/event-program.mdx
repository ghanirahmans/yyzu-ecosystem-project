# Alur Pelaksanaan Program (Event/Program Workflow)

Dokumen ini menjelaskan alur kerja dan kalender pelaksanaan program di ekosistem YYZU. Alur ini disusun untuk memastikan setiap kegiatan pembelajaran berjalan teratur, terpantau, dan memberikan dampak nyata terhadap pengembangan kompetensi anggota.

Aktivitas YYZU dapat mencakup:
- study group,
- mentoring session,
- workshop,
- technical discussion,
- product discussion,
- collaborative project,
- mini project,
- code review,
- design review,
- portfolio review,
- workflow simulation,
- open-source contribution,
- internal product development,
- real-world project development,
- industry sharing,
- dan career preparation.

## Alur Pelaksanaan Program

Setiap kegiatan atau program kerja YYZU wajib mengikuti alur kerja sistematis berikut untuk memastikan kualitas pelaksanaan yang tinggi:

```
Idea ──> Objective ──> PIC ──> Planning ──> Publication ──> Execution ──> Documentation ──> Evaluation
```

1. **Idea (Ideasi Kegiatan)**: Pengumpulan usulan program dari member, pengurus, atau berdasarkan evaluasi kebutuhan ekosistem. Ide kemudian diajukan dalam rapat kepengurusan untuk disaring kelayakannya.
2. **Objective (Tujuan & Parameter)**: Menentukan tujuan utama kegiatan secara spesifik, mengidentifikasi target audiens yang dituju, serta menetapkan indikator kesuksesan acara.
3. **PIC (Penunjukan Penanggung Jawab)**: Menunjuk satu orang penanggung jawab utama (*Person in Charge*) dari tim *Event Organizer* yang akan memimpin koordinasi persiapan hingga akhir acara.
4. **Planning (Perencanaan Operasional)**: Penyusunan jadwal, pembuatan linimasa persiapan (*timeline*), penentuan pembicara/moderator, penyusunan rancangan anggaran (jika ada), serta pembagian tugas kepanitiaan teknis.
5. **Publication (Publikasi Konten)**: Tim *Event Organizer* bekerja sama dengan *Media & Branding* untuk memproduksi aset visual promosi (poster, brosur, feed) dan menyebarluaskan informasi acara ke kanal komunikasi komunitas serta publik.
6. **Execution (Pelaksanaan Acara)**: Penyelenggaraan acara sesuai dengan jadwal dan susunan acara (*rundown*) yang disepakati. Seluruh panitia bekerja sama mengamankan kelancaran teknis.
7. **Documentation (Pengarsipan & Laporan)**: Menyusun laporan singkat pasca-acara yang memuat data kehadiran peserta, tautan rekaman sesi (jika ada), materi presentasi pembicara, serta arsip foto/video dokumentasi.
8. **Evaluation (Sesi Evaluasi)**: Menyelenggarakan rapat evaluasi untuk membahas apa saja kendala yang terjadi, bagaimana respons peserta, serta menyusun daftar tindakan perbaikan untuk pelaksanaan program di masa mendatang.
